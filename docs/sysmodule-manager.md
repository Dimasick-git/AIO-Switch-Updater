# Управление sysmodules

Sysmodule (от system module) — фоновый процесс, который Atmosphere
запускает поверх HOS. Это всё, что лежит в `/atmosphere/contents/<tid>/`:
Sys-clk, ovlSysmodules, Mission-Control, FPSLocker и так далее.

## Что делает менеджер

`Tools → Управление sysmodules` показывает все папки `contents/<tid>/`
с переключателем «Вкл/Выкл» для каждой.

Под капотом — единственный примитив: файл `flags/boot2.flag` внутри
папки сабмодуля.

- **Файл есть** → Atmosphere стартует модуль при загрузке.
- **Файла нет** → модуль присутствует на SD, но не загружается.

То есть «выключение» НЕ удаляет .kip и НЕ ломает обновления.
Включение обратно — создание пустого файла.

Изменения вступают в силу **после перезагрузки**. Sysmodule не
дёргается в рантайме.

## Какое имя показывается

Менеджер пытается прочитать `toolbox.json` в папке модуля, и если там
есть поле `"name"` — берёт его. Иначе подставляет TitleID
(шестнадцатеричный, 16 символов).

```json
{
  "name": "ovlSysmodules",
  "tid":  "420000000007e51a",
  ...
}
```

## Какие модули нельзя выключить

Семь системно-критичных TID помечены как `[SYS]` и переключение для них
запрещено:

| TitleID            | Модуль       | Что сломает выключение |
|--------------------|--------------|------------------------|
| `0100000000000000` | fs           | Никакая файловая система не загрузится |
| `0100000000000010` | sm           | Service Manager — все IPC откажут |
| `0100000000000034` | loader       | Не запустит ни одного процесса |
| `0100000000000037` | ncm          | Не найдёт ни одного содержимого |
| `0100000000000036` | pm           | Никакого process management |
| `010000000000000d` | boot         | Не дойдёт до меню |
| `010000000000004d` | ams_mitm     | Atmosphere mitm — без него нет sigpatches |

Если попробовать включить toggle на одном из этих модулей — появится
диалог «Системно важный модуль. Переключение запрещено.» Менеджер
честно говорит: чтобы выключить fs, вам нужно либо вручную
переименовать папку, либо обратиться в `/aio-backup/` и подумать, что
вы делаете.

## Что делать если что-то сломалось

После toggle’а у вас на `/aio-backup/` НЕ создаётся снимок — это
лёгкая операция, точечный flag. Если конкретный sysmodule всё же
сломал систему, варианты:

1. Загрузиться в Hekate → Tools → SD card управление → удалить
   `flags/boot2.flag` соответствующего модуля.
2. Если SD не читается — Hekate → emuMMC / прошить через UMS.

Это абсолютно ортогонально функции «Восстановление /atmosphere/» —
там перезаписывается весь /atmosphere, а здесь меняется один байт.

## Связанные модули

- `Tools → Восстановление /atmosphere/` — крупная операция, см.
  `docs/factory-restore.md`.
- `About → Состояние CFW` — после включения сомнительного модуля
  посмотрите, не упало ли количество sigpatches или contents.
