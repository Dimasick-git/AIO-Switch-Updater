# Политика безопасности

## Поддерживаемые версии

Поддерживается **только последний релиз** (`releases/latest`). Старые
сборки не обновляются — если уязвимость найдена в более ранней сборке,
сначала обновитесь до текущей и проверьте, воспроизводится ли проблема.

## Как сообщить об уязвимости

**Не открывайте публичный issue.** Возможные каналы:

1. **GitHub Security Advisory** — на странице
   <https://github.com/Dimasick-git/AIO-Switch-Updater/security/advisories/new>.
   Это самый правильный путь — переписка зашифрована и видна только
   мейнтейнеру.
2. **Telegram-канал** [@Ryazhenkacfw](https://t.me/Ryazhenkacfw) —
   написать мейнтейнеру в личку через канал.

Что включить в репорт:

- Версия Ryazhenka Updater (тег релиза или из About tab).
- Минимальные шаги воспроизведения.
- Влияние (RCE / DoS / leak / privilege escalation).
- Если есть PoC — приложите, ссылка на gist приветствуется.

## Что считается уязвимостью

- Удалённое выполнение кода через подмену скачиваемых архивов.
- Запись/чтение произвольных файлов вне `/switch/aio-switch-updater/`
  и `/config/aio-switch-updater/`.
- Кража токенов из `/config/aio-switch-updater/token.json`.
- Bypass проверок целостности (SHA256, NACP).

## Что не считается уязвимостью

- Падения от испорченных файлов NACP — приложение их явно warn'ит.
- HTTP downgrades на сторонних серверах (мы не контролируем GitHub /
  Mega / Cheat Slips).
- Race conditions в апстримном коде — репортите в
  <https://github.com/HamletDuFromage/aio-switch-updater/security>.
